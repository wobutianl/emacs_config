#+TITLE: path

* 我想知道的事
** 怎么在不同图层间算路？
* path数据给谁
- 流程：
  - data -> path -> guide(诱导） + location（定位） + map（地图显示） + search
* 算法
- dijkstra
- A*
- HH
- landmark
- 主要考虑各种因素与其相关的权重
* path主要工作
- 因为算法相对固定，所以可以不考虑
- 考虑品质（就是考虑哪些因素 与 权重的取值）
- 量化cost种类
  - 道路 link cost ：长度 + 种类 + 宽度。。。。
  - 道路 link turn cost : 红绿灯 + 道路种别切换 。。。
* HH分层探索
- 数据基础 
  - parcel 数据 
  - region （专为path服务）
    - 不同层次数据进行了多层的抽象（多层搜索的前提）
- 过程
  - 起点与目标点 同时进行parcel（level0)的探索->得到region点（或者直接找到路了）
  - 提升到region层去搜索（同时进行上层迁移搜索）
  - 一层层的搜索，直到两端相遇
- 二点探索
