#+TITLE: 赶集架构知识点

* kafka
- linkedin 用于日志处理（分布式消息队列）
- 用户行为 + 系统运行日志
* zookeeper
- hadoop 针对大型分布式系统的可靠协调系统
- 配置 + 名字 + 分布式同步 + 组 服务
- 封装好易出错的关键服务
* broker
- 分布式模式之一：动态改变，添加，删除服务端
- client, server, client_proxy, server_proxy, broker, bridge
* Q4M
- 基于Mysql的轻量级消息队列，通过扩展SQL来操作消息队列
- 支持多方发送，多方接收，多队列
* fastDFS
- 分布式文件系统，文件存储，文件同步，文件访问
- 大容量存储，负载均衡
- 适合（图片，视频网站）
* NFS：network,file system
- 网络文件系统，允许用户和程序可以在本地访问远端系统上的文件
* Xapian
- 全文检索程序（C++）
- （Java）Luceue
* CDC：设备上下文类
* 简称
- PV：planned value:应该完成多少工作
- EV:earned value:完成了多少工作
- AC:actual cost：实际成本
- BAC:budget cost at completion：完成预算
- EAC:estimate at completion：完成成本
- ETC:剩余工作
* scribe
- facebook 日志收集系统
- 可以从不同的数据源，机器中收集日志
* flume
- cloudera 海量日志采集，聚合，传输系统
- 日志收集，数据处理
* netscaler
- 优化web应用交付的综合解决方案
- web应用加速，数据中心效率，保护信息，集中化管理
* nginx（engine x)
- HTTP、反向代理服务器
- Apache的代替品
* SSO : ;single sign on
- 单点登录：在多个应用系统中，只要登陆一次，就可以访问相互信任的应用系统
- 企业业务整合方案
* EMP
- 企业管理平台
